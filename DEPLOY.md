# Перенос бота на сервер/ПК (Linux, 24/7)

Цель — бот работает круглосуточно сам, без окна и без чужого ПК.

## Что нужно от машины
- **Linux** (Ubuntu Server 22.04, без графики — GUI не нужен).
- **4–8 ГБ RAM** (8 комфортно), **~20 ГБ диска**, интернет. GPU не нужен.
- Доступ по SSH под **root** (или пользователь с `sudo`).

## Установка — 3 команды

```bash
git clone https://github.com/kokolsk1y/operator-card-bot.git
cd operator-card-bot
sudo bash deploy.sh
```

Скрипт сам:
1. поставит Docker (если нет);
2. спросит **токен бота** (вставляешь один раз → сохранится в `.env`);
3. соберёт образ (Node + Python + rembg + модель birefnet + chromium — всё внутри) и запустит.

Первая сборка ~10–15 мин (качается модель ~1 ГБ). Потом бот работает сам.
Проверь в Telegram: `/start`.

## Управление

```bash
sudo docker compose logs -f      # логи в реальном времени
sudo docker compose restart      # перезапуск
sudo docker compose down         # остановить
sudo docker compose up -d --build   # обновить после git pull
git pull && sudo docker compose up -d --build   # обновить код + перезапустить
```

- `restart: unless-stopped` (в `docker-compose.yml`) = бот сам поднимается после сбоя **и после перезагрузки сервера**.
- Готовые карточки / вырезанные фото / обрезки складываются в папки на сервере (примонтированы наружу контейнера).

## Важно
- Токен **не в образе и не в git** — только в `.env` на сервере.
- **Один сервер = один экземпляр** бота. Параллельно на другом ПК запускать НЕЛЬЗЯ (конфликт Telegram 409). Перед запуском на сервере останови бота на старом ПК (`pm2 stop operator-bot` на Windows).
- Модель вырезки — **birefnet** (качество). На CPU ~20–40 сек/фото; вырезка идёт в фоне, оператор не ждёт.

## Если сервер слабый / много операторов разом
Вырезка фото прожорлива по памяти. При 8 ГБ и очереди — норм. Если будут «падения по памяти» —
скажи, добавлю ограничение одновременных вырезок (2–3 за раз, остальные ждут пару секунд).

## Ручная установка (без Docker) — если понадобится
Node.js 20+, Python 3 (+ `pip install -r requirements.txt`), `npx puppeteer browsers install chrome`,
токен в `config.json` или env `BOT_TOKEN`, запуск `node index.mjs` под pm2/systemd.
