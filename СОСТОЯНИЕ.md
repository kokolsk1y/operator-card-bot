## 1. Что бот умеет (v2)

Генератор карточек товаров 900×1200 (**3:4**, Озон принимает как есть).

- `/start` — показывает **пример-эталон** карточки + начинает.
- `/new` — новая карточка. `/help` — инструкция. Команды есть в меню Telegram.
- **`/crop` (или `/обрезка`)** — отдельный режим: фото товара на белом → обрезка под **3:4** для Озон
  (товар по центру, лишний фон обрезан, само фото не искажается). Файлы → папка `обрезка 3-4/`. Скрипт `crop34.py`.
- Шаги: **название → бейдж(опц.) → фото → до 4 характеристик → «Собрать карточку»**.
- **Фото обрабатывается в фоне** — оператор сразу пишет характеристики, не ждёт вырезку.
- Кнопки под готовой карточкой: **🔄 Заголовок · 🖼 Фото · 📝 Характеристики · 🔆 Свечение · 📐 Подвинуть · 🆕 Новая**.

## 2. Формат ввода

- **Разделитель строк — `^` или перенос строки** (НЕ «/», чтобы «16/250 В» не ломалось).
- Заголовок: `Удлинитель ^ с выключателем` (или просто текст — 1-е слово крупное).
  Заголовок + подзаголовок **по центру**, с боковыми отступами.
- Характеристика: `значение ^ подпись ^ примечание`, напр. `10А ^ нагрузка ^ *до 2300Вт`.
  Без `^` — одна строка (`3 розетки`). 1-я характеристика → оранжевый квадрат, остальные → полоски с линией-разделителем.
- Лимит **20 символов** на часть + **авто-фит** (шрифт ужимается, за зону не вылезет).
- Если характеристик **нет** — товар встаёт **по центру** карточки.

## 3. Свечение вокруг товара

- **Авто по яркости товара**: `cutout.py` считает долю светлых пикселей → белый товар = **слабое** свечение, тёмный = сильное.
- **Вручную** — кнопка **🔆 Свечение**: цикл `авто → нет → слабое → среднее → сильное`, пересборка сразу.
- 4 уровня в `render.mjs` (объект `GLOW`).

## 4. Сейф-зоны (товар не перекрывает текст)

Товар = `contain`+центр внутри **динамически вычисленной зоны**:
- **Верх товара** = низ шапки (заголовок ~180 / подзаг ~238 / бейдж ~322 — что присутствует) **+ 28px**.
- **Гориз.**: есть характеристики → правее колонки (край 260 +40 = слева 300); нет → по центру (≤660 шириной).
- Низ = 1160. Гарантия: фото не заходит на заголовок/бейдж/характеристики при любом контенте.

## 5. Режим «📐 Подвинуть» + масштаб

- Стрелки двигают выбранный элемент (◀▶ — выбор: заголовок / бейдж / оранж.блок / вся группа / фото / каждая строка).
- Шаг 5/10/20px. **🔍➖/🔍➕ Фото** — масштаб товара 50–200%. Живой пересбор через editMessageMedia.

## 6. Технологии

| Слой | Что |
|---|---|
| Бот | Node.js + **grammY** (polling) |
| Вырезка фона | Python + **rembg / birefnet-general** (модель ~1 ГБ в `~/.u2net/`) |
| Рендер | **HTML + Puppeteer(chromium) → PNG**, ресайз Sharp до 900×1200 |
| Шрифт | **Montserrat ExtraBold (800)** из `@fontsource` (латиница+кириллица) |
| Цвет текста | `#F2EBEB` · фон синий (в `фон.png`) · оранжевый `#E0643C` |

**Рендер изолирован в подпроцесс** `render-worker.mjs` — если puppeteer падает, бот выживает.
Успех сборки = наличие файла (даже если chromium капризит на закрытии).

**Очередь тяжёлых операций** (защита от нехватки RAM при многих операторах): не более N вырезок фона
и рендеров одновременно, остальные ждут. Настройка — `concurrency` в config (по умолч. cutout:2, render:1).

## 8. Запуск и эксплуатация (сейчас — на ПК)

- Бот держит **pm2** (демон, независим от терминала): `pm2 restart operator-bot`, `pm2 logs operator-bot`, `pm2 stop operator-bot`.
- **Автозапуск** при входе в Windows: `C:\Users\ikoko\.pm2\autostart.cmd` в папке Автозагрузки (vbs).
- Работает, **пока включён ПК**. Только ОДИН экземпляр (иначе конфликт 409).
- Токен: в `config.json` (локально) или переменная `BOT_TOKEN` (на сервере).

⚠️ **Важно про chrome:** puppeteer'у нужен установленный chrome. Если пропал («Could not find Chrome») —
`cd` в корень проекта и `npx puppeteer browsers install chrome`. При `npm install` ставится сам (postinstall).
