## 1. Суть задачи (от Сергея)

Операторы магазина (кабинет Зверев) делают карточки товаров для маркетплейса.
Нужен бот, который из минимума ввода собирает готовую карточку:

1. **Создать бота** (Telegram).
2. **Фон фиксированный** — оператор его не выбирает, бот всегда берёт `фон.png`.
3. Оператор пишет **название товара** → крупный заголовок сверху.
4. Оператор прикладывает **фото товара** → бот **сам** вырезает фон + добавляет
   лёгкое свечение по краю (чтобы края не были «кривыми»).
5. Бот отдаёт **готовую карточку** (PNG).
6. (желательно) **главные характеристики** слева:
   - до 4 штук;
   - 1-я — «главная», в **оранжевом квадрате**;
   - остальные — **полоски**;
   - незаполненные **не выводятся**;
   - **лимит по буквам в строке** — чтобы текст не переносился и не вылезал за свою зону.

Образец макета, который хочет получить Сергей: `пример карточки.png`.
Фон: `фон.png` (900×1200).

Пример ввода характеристик от оператора:
```
Характеристика №1 - 1,5 метра      (главная, оранжевый квадрат)
Характеристика №2 - ПВС 3х1 мм      (полоска)
Характеристика №3 - 10А нагрузка    (полоска)
Характеристика №4 - 3 розетки       (полоска)
```

## 2. Стек (решено)

| Слой | Выбор | Почему |
|---|---|---|
| Интерфейс | **Telegram-бот**, режим polling | Бесплатно, с телефона, без сервера. Запуск с ПК. |
| Язык/рантайм | **Node.js** (grammY) | Один язык с проектом, переиспользуем куски. |
| Удаление фона | **rembg + birefnet-general.onnx** (Python) | Лучшее качество. Модель ~1 ГБ уже скачана в `~/.u2net/`. |
| Свечение по краю | **Sharp** (Node) | Размыть силуэт → белый ореол под товаром. |
| Рендер карточки | **HTML-шаблон + Puppeteer → PNG** | 1-в-1 с примером, CSS легко тянет зоны/полоски/авто-фит. |

**Старт — бесплатно и локально.** Расширение (v2): webhook + VPS, очередь, несколько операторов, SQLite, пресеты брендов.

## 3. Измерения макета (обмер `пример карточки.png`, 827×1100)

Все зоны переведены в проценты — чтобы корректно лечь на фон 900×1200.
`scale: X×1.088, Y×1.091`.

| Элемент | Пример (px) 827×1100 | Доля | Цель (px) 900×1200 |
|---|---|---|---|
| **Фон синий** | весь | — | `RGB(54,66,154)` ≈ `#3642A0` |
| **Оранжевый** | — | — | `RGB(224,100,60)` ≈ `#E0643C` |
| **Треугольник** (верх-лево) | ширина 395, высота 206 | 47.7% × 18.7% | ≈430 × 224 |
| **Заголовок (1 строка)** | y 75–149, выс. ~75 | font-cap ≈7% H | font ≈ 95–100px |
| **Подзаголовок** | y 172–201, выс. ~30 | | font ≈ 36px |
| **Бейдж «с заземлением»** (рамка) | y ≈ 225–290, по центру | | опционально |
| **Левый отступ колонки** | x = 38 | 4.6% | ≈ 42 |
| **Оранжевый квадрат** (главная хар-ка) | x 38–196, y 295–417 → **159 × 123** | 19.2% × 11.2% | ≈ 173 × 134 |
| **Полоса 1 (ПВС)** | крупн. y 474–515; мелк. y 529–553 | | |
| **Полоса 2 (10А)** | крупн. y 600–649; мелк. y 659–683; примеч. y 696–709 | | |
| **Полоса 3 (3 розетки)** | y ≈ 745+ | | |
| Шаг между полосами | ≈ 125 px | ≈ 11.4% H | ≈ 137 |
| **Граница текст\|товар** | x ≈ 200 | 24.2% | ≈ 218 |
| **Зона товара** | x 200–826 (вся высота) | правые 76% | x 218–900 |

Левая колонка (заголовок, оранжевый квадрат, полосы) **лево-выровнена** по x≈42 (5%).
Товар центрируется в правой зоне, по вертикали — по центру/чуть ниже.

> ⚠️ Финальные пиксели подгоняются «живым» рендером: сгенерили → наложили на пример → сверили → поправили. Цифры выше — стартовая сетка.

## 4. Текст без переносов («лимит по буквам в строке») — решение

Сергей: лимит не по словам, а **чтобы строка не переносилась и не вылезала за зону**.

**Двойная защита:**
1. **Мягкий лимит** на вводе: > ~20 символов → бот пишет «слишком длинно (N/20), сократи» и ждёт заново. Значение лимита подбирается под реальную ширину зоны.
2. **Жёсткий авто-фит** на рендере: у каждого текстового блока **фиксированная ширина-зона**; размер шрифта **автоматически уменьшается**, пока строка не влезет в одну линию. Гарантия, что НИКОГДА не вылезет, даже если лимит обошли.

Так зоны не «скачут»: блоки стоят на фиксированных Y, текст ужимается внутри своего бокса.

## 5. Удаление фона — исследование (выполнено)

Тест на «грязном» фото (серый товар на белом, тонкие лапки + сквозные отверстия):

| Модель | Время | semi% (кайма) | Отверстия | Вердикт |
|---|---|---|---|---|
| isnet-general-use | **1.2 с** | 0.89 | видит | быстро, но снизу лёгкая кайма |
| **birefnet-general** | 14.9 с | **0.69** | **видит чисто** | **лучшее качество** ← выбор |
| birefnet + alpha-matting | 14.9 с | 0.69 | — | без разницы на этом фото |

**Решение:** по умолчанию **birefnet-general** (качество — приоритет, это самое трудное место).
~15 с/фото на CPU — приемлемо, бот пишет «обрабатываю…».
Опция «быстро» (isnet) — на потом.

**Проверка результата (обязательна, по просьбе):** после вырезки бот считает метрики
(площадь товара %, доля полупрозрачной каймы, «дыры») и:
- авто-подчищает край (decontaminate) + обрезает по tight-bbox;
- если качество подозрительное (товар занял <5% или >95%, рваный силуэт) — **показывает превью и просит переснять/прислать другое фото**.

Всегда показываем оператору превью вырезанного товара ДО финальной карточки — кнопка «переснять фото».

## 5.1. Формат ввода (двухуровневые тексты, как в примере) — уточнение v1.1

Пример Сергея обмерян по пикселям. Карточка точно повторяет его:
- **Заголовок двухуровневый**: большое слово + подзаголовок мельче. Ввод:
  `Удлинитель / с выключателем` (через «/») или просто `Удлинитель с выключателем`
  (тогда 1-е слово — большое, остальное — подзаголовок). Большое слово — ЗАГЛАВНЫМИ.
- **Бейдж в рамке** (опц.): отдельный шаг, напр. `с заземлением`.
- **Характеристики двухстрочные + линия-разделитель** под каждой. Ввод через «/»:
  `значение / подпись / примечание`
  - `1,5 / метра` (главная → оранжевый квадрат)
  - `ПВС / 3х1 мм`
  - `10А / нагрузка / *до 2300Вт`
  - `3 розетки` (без «/» → одна строка)

Применённые размеры шрифтов @900 (из обмера): заголовок 120 / подзаг. 48 (ЗАГЛАВНЫМИ) / бейдж 40 /
оранж. 84+32 / характеристика 66+40 / примечание 22. Линии: ширина 172, толщина 3px.
Всё с **авто-фитом** (ужимается, если длиннее зоны). Все числа — в объекте `LAYOUT` в `render.mjs`.

**Шрифт: Montserrat ExtraBold (вес 800)** — латиница+кириллица из `node_modules/@fontsource/montserrat/files/`
(сабсеты `*-latin-800` и `*-cyrillic-800` через unicode-range). Bold 700 был тоньше эталона — заменён на 800.
Подзаголовок заголовка рендерится ЗАГЛАВНЫМИ (как «С ВЫКЛЮЧАТЕЛЕМ» в примере); бейдж и вторичные строки — как набрано.

## 6. Алгоритм работы бота (FSM-диалог)

```
/new  (или /start → «Сделать карточку»)
 │
 ├─ Шаг 1. «Пришли НАЗВАНИЕ товара»
 │         → заголовок сверху (авто-фит, перенос на 2 строки разрешён)
 │
 ├─ Шаг 2. «Пришли ФОТО товара»
 │         → rembg(birefnet) вырезает фон
 │         → Sharp добавляет свечение
 │         → проверка качества + tight-crop
 │         → показать превью товара  [✅ ок] [🔁 переснять]
 │
 ├─ Шаг 3. «Главная характеристика (≤20 симв., или “-” пропустить)»  → оранжевый квадрат
 ├─ Шаг 4. «Характеристика №2 (≤20, “-” пропустить)»                 → полоска
 ├─ Шаг 5. «Характеристика №3 (≤20, “-” пропустить)»                 → полоска
 ├─ Шаг 6. «Характеристика №4 (≤20, “-” пропустить)»                 → полоска
 │         (на каждом шаге: >20 симв → «сократи», пустые → не выводятся)
 │
 └─ СБОРКА: фон + заголовок + полосы + товар(свечение) → Puppeteer → PNG
            → отправить карточку
            → кнопки: [🔄 Заголовок] [🖼 Фото] [📝 Характеристики] [✅ Готово]
               (любую часть можно пересобрать без начала с нуля)
```
